Core concept (project-specific)

hyper-mcp is an MCP server that exposes ScoutOS-style ports (data / cache / blob / queue / search / auth) over the Model Context Protocol. It runs on two transports:

  • stdio (local/dev) — no HTTP bearer-token step
  • StreamableHTTP (hosted) — JWT-verified, account-scoped, with an internal capability-RPC boundary for Daytona sandboxes

The security model relies on jose for JWT verify/sign, scrypt for password hashing, AsyncLocalStorage for request-scoped auth, and an adapter pattern (VM prototype vs Daytona sandbox) for function execution. The prod app_data layer uses Postgres row-level security with FORCE ROW LEVEL SECURITY.
